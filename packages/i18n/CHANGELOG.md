# Changelog

All notable changes to `@brika/i18n` are documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.1.0/),
and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [Unreleased]

### Changed

- Public types narrowed to remove all `as` casts: loaders, fallback chain,
  registry, and React hooks all use type guards or zod schemas at every
  boundary.
- React entry split into 3 hooks (`useTranslate`, `useIntl`, `useLocale`) plus
  a unified `useLocale()` façade. `TFunction` is augmented via declaration
  merging to accept the i18next `(key, options)` shape.
- `useLocale().tp(namespace, key, defaultValue?, __cs?)` now forwards an
  optional `__cs` (call-site marker) into the options bag so build-time
  source-location capture works for namespaced lookups.

### Added

- `@brika/i18n/node` exports: `loadLocaleFolder`, `loadMergedLocaleFolder`,
  `detectFileIndent`, `detectIndentFromContent`, `watchLocaleSource`,
  `findWorkspaceRoot`, `discoverPackageLocales`, `PackageJsonSchema`.
- `@brika/i18n/react` exports: `useTranslate`, `useIntl`, `useLocale`,
  `I18nProvider`, `switchLanguage`.
- `@brika/i18n/registry` module augmentation entry for declaration-merged
  namespace types generated by `@brika/i18n-devtools`.
- Closure-encapsulated `I18nClient` — no module-scoped mutables.

### Security

- All JSON loads sanitize unsafe key segments (`__proto__`, `constructor`,
  `prototype`) before reaching the runtime registry.
- `setNestedValue` rejects unsafe segments at every depth.

## [0.3.0] — 2026-05-21

Initial public-shape release as a standalone, framework-agnostic package. No
brika-specific concepts in the runtime — works in any React + Vite + Bun project.
