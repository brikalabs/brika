## Primary

Solid fill. Use for the main call-to-action on a page or dialog.

## Outline

Lower-emphasis secondary action that still needs clear affordance.

<ComponentExample code={`<Button variant="outline">Cancel</Button>`}>
  <ButtonOutlineDemo client:load />
</ComponentExample>

## Destructive

Irreversible actions. Always pair with a confirm step.

## Props

| Prop      | Type                                                                  | Default   |
| --------- | --------------------------------------------------------------------- | --------- |
| `variant` | `default \| destructive \| outline \| secondary \| ghost \| link`     | `default` |
| `size`    | `default \| xs \| sm \| lg \| icon \| icon-xs \| icon-sm \| icon-lg`  | `default` |
| `asChild` | `boolean` — project props onto the first child via Radix `Slot.Root`  | `false`   |
| ...native | All native `<button>` attributes (`onClick`, `type`, `disabled`, …)   | —         |

## Accessibility

- Focus-visible rings use the `--ring` token for WCAG contrast.
- `disabled` reduces opacity and blocks pointer events.
- Icon-only buttons REQUIRE an `aria-label`.
- `asChild` lets you apply button styles to a link without nesting — keeps the native element semantic.

## Tokens consumed

`primary`, `primary-foreground`, `destructive`, `secondary`, `accent`,
`background`, `ring`, `border`. See [button.tokens.ts](https://github.com/brikalabs/brika/blob/main/packages/clay/src/components/button/button.tokens.ts).
