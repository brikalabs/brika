## Icon-only


## Variants

| Variant        | When to use                                                |
| -------------- | ---------------------------------------------------------- |
| `default`      | Primary call-to-action.                                    |
| `outline`      | Secondary action next to a primary one.                    |
| `secondary`    | Mid-emphasis actions, grouped toolbars.                    |
| `ghost`        | Low-emphasis actions in dense UIs.                         |
| `link`         | Actions that look like links but behave like buttons.      |
| `destructive`  | Irreversible actions — always pair with a confirm dialog.  |

## Sizes

`xs`, `sm`, `default`, `lg` for text buttons. `icon-xs`, `icon-sm`, `icon`,
`icon-lg` for square icon-only variants.

## Accessibility

- Focus-visible rings use the `--ring` token for WCAG contrast.
- `disabled` reduces opacity and blocks pointer events.
- Icon-only buttons REQUIRE an `aria-label`.
