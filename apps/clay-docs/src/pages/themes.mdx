    <code class="rounded border border-clay-hairline bg-clay-base px-1 py-px font-mono text-[0.8125rem] text-clay-default">applyTheme(theme, mode)</code> writes every colour onto <code class="rounded border border-clay-hairline bg-clay-base px-1 py-px font-mono text-[0.8125rem] text-clay-default">document.documentElement</code> as a CSS custom property.   </p>

  <p class="mb-5 max-w-2xl text-clay-default text-sm leading-relaxed">
    <code class="rounded border border-clay-hairline bg-clay-base px-1 py-px font-mono text-[0.8125rem] text-clay-default">themeToCssVars(theme, mode)</code> returns a <code class="rounded border border-clay-hairline bg-clay-base px-1 py-px font-mono text-[0.8125rem] text-clay-default">CSSProperties</code> object you can spread onto any element's style prop. The theme applies only to that subtree — useful for side-by-side comparisons like the gallery above.
  </p>
  <CodeSource
    code={`import { ocean, themeToCssVars } from "@brika/clay/themes";

<div style={themeToCssVars(ocean, "light")}>
  <Button>Ocean button</Button>
</div>`}

    filename="scoped"

  <h2 id="persist" class="mb-3 scroll-mt-24 font-display text-3xl text-clay-strong leading-[1.1]">
    Remembering the choice
  </h2>
  <p class="max-w-2xl text-clay-default text-sm leading-relaxed">
    Clay stores the active theme id under the <code class="rounded border border-clay-hairline bg-clay-base px-1 py-px font-mono text-[0.8125rem] text-clay-default">clay-theme</code> key in <code class="rounded border border-clay-hairline bg-clay-base px-1 py-px font-mono text-[0.8125rem] text-clay-default">localStorage</code>. The picker reads it on mount and re-applies the selection. If storage is unavailable, the theme falls back to <code class="rounded border border-clay-hairline bg-clay-base px-1 py-px font-mono text-[0.8125rem] text-clay-default">default</code> for the session.
  </p>
